# Koda CLI Configuration

## О проекте

**NoteTracker** — PWA приложение для отслеживания нот и музыкальных упражнений.

### Технологии
- React + TypeScript
- Vite
- Bootstrap CSS + Material Design
- PWA (Service Worker, Web App Manifest)

### Структура проекта

```
src/
├── App.tsx              # Главный компонент
├── components/          # UI компоненты
│   ├── DayTabs.tsx      # Вкладки дней
│   ├── ExerciseCard.tsx # Карточки упражнений
│   ├── Modals.tsx       # Модальные окна
│   └── SettingsPanel.tsx# Панель настроек
├── hooks/               # Кастомные хуки
│   ├── useLocalStorage.ts
│   └── useTheme.ts
└── utils/               # Утилиты
    ├── constants.ts
    └── helpers.ts
```

## Команды разработки

```bash
# Установка зависимостей
npm install

# Запуск dev-сервера
npm run dev

# Сборка для продакшена
npm run build

# Запуск линтинга
npm run lint

# Предпросмотр продакшена
npm run preview
```

## Соглашения проекта

### Именование
- Файлы компонентов: `PascalCase.tsx`
- CSS-файлы компонентов: `ComponentName.css`
- Хуки: `useXxx.ts`
- Утилиты: `kebab-case.ts` или `snake_case.ts`

### Код-стайл
- TypeScript с строгой типизацией
- Functional компоненты с hooks
- CSS модули или локальные стили

## Особенности Koda CLI

### Безопасность
- Не коммитьте секреты и API-ключи
- Подтверждайте критические изменения

### Рабочий процесс
1. Понимание задачи через анализ кода
2. Планирование изменений
3. Реализация с соблюдением соглашений
4. Проверка через тесты и линтер

### Коммиты
- Сообщения на русском или английском
- Акцент на "почему", а не "что"
- Формат: `feat: описание`, `fix: описание`, `chore: описание`

### Правила
- Не лезть в другие классы/компоненты, о которых не говорилось
- Не ломать логику приложения/сайта
- Думать и анализировать, не поломают ли действия приложение
- Не удалять и не изменять классы/теги/JS-код, о которых не говорилось, если это ломает логику
- `/no_think` — отвечать кратко, сразу переходить к делу
